Upload Files

PUT /api_upload/

Upload a CSV file for translation. The file will be processed asynchronously, and you can check its status later.

Request
Response

Headers

Name Type Description
Authorization string Bearer token from authentication
Content-Type string text/csv or application/octet-stream
Content-Length number Size of the file in bytes

Body

The raw CSV file content as the request body.

Example with cURL 

curl -X PUT \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: text/csv" \
  --data-binary "@/path/to/your/file.csv" \
  https://api.example.com/upload/yourfile.csv

Response 

{
    "message": "File uploaded successfully",
    "filename": "yourfile contents",
    "status": "complete",
    "timestamp": "2023-06-15T12:34:56Z"
}

Get File Content

GET /files/{filename}

Retrieve the content of a specific translated file. The response will be the raw CSV content.

Request
Response

Headers

Name Type Description
Authorization string Bearer token from authentication
Accept string text/csv (default) or application/json

Example with cURL 

curl -X GET \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Accept: text/csv" \
  https://api.example.com/files/translated_file1.csv

Response

CSV content:

id,original_text,translated_text
1,"Hello world","Bonjour le monde"
2,"Good morning","Bonjour"

Or JSON format (when Accept: application/json is specified):

{
    "filename": "translated_file1.csv",
    "content": [
        {
            "id": "1",
            "original_text": "Hello world",
            "translated_text": "Bonjour le monde"
        },
        {
            "id": "2",
            "original_text": "Good morning",
            "translated_text": "Bonjour"
        }
    ]
}

API Examples

Using Postman

Here's how to use the API with Postman:

  1. First, authenticate to get your token
  2. Create a new request for the endpoint you want to use
  3. Set the Authorization header to "Bearer YOUR_TOKEN"
  4. For file uploads, use the binary body type and select your file

Using cURL

Here's a complete workflow using cURL:

# 1. Authenticate
TOKEN=$(curl -X POST \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"yourpassword"}' \
  https://api.example.com/auth/login | jq -r '.token')

# 2. Upload a file
curl -X PUT \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: text/csv" \
  --data-binary "@/path/to/file.csv" \
  https://api.example.com/upload/file.csv

# 3. List files
curl -X GET \
  -H "Authorization: Bearer $TOKEN" \
  https://api.example.com/files

# 4. Download a file
curl -X GET \
  -H "Authorization: Bearer $TOKEN" \
  -o downloaded_file.csv \
  https://api.example.com/files/file.csv
