Authentication

POST /api_keys

Generate an API key to authenticate your requests.

Request

{
    "Authorization": "Bearer [ID_TOKEN]",
    "Content-Type": "application/json"
}

Response

{
    "api_key": "your_api_key_here",
    "limits": {
        "rate_limit": "10",
        "burst_limit": "20",
        "daily_quota": "1000"
    }
}

Include the API key in all requests:

Authorization: Bearer your_api_key_here

Upload Files

PUT /api_upload/

Upload a CSV file for translation.

Example Request

curl -X PUT \
  -H "Authorization: Bearer API_KEY" \
  -H "Content-Type: text/csv" \
  --data-binary "@/path/to/file.csv" \
  https://api.example.com/api_upload/

Response

{
    "message": "File uploaded successfully",
    "filename": "yourfile.csv",
    "status": "processing"
}

List Files

GET /files

List all available translated files.

Response

{
    "files": [
        {
            "filename": "file1.csv",
            "size": 1024,
            "status": "completed"
        }
    ]
}

Get File Content

GET /files/{filename}

Download a translated file.

Response (CSV)

id,original_text,translated_text
1,"Hello","Bonjour"

Examples

cURL Workflow

# Generate API Key
TOKEN=$(curl -X POST -H "Authorization: Bearer $ID_TOKEN" ...)

# Upload file
curl -X PUT -H "Authorization: Bearer $TOKEN" ...

# List files
curl -X GET -H "Authorization: Bearer $TOKEN" ...
